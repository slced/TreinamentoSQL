ESPECIFICAÇÕES

São códigos SQL que você irá definir o tipo de dado que você estará movimentando.

NUMÉRICOS

  • TINYINT = Geralmente armazena 1 byte (8 bits), permitindo representar números inteiros no intervalo de -128 a 127 (ou 0 a 255 se for sem sinal).

  • SMALLINT = Geralmente armazena 2 bytes (16 bits), permitindo representar números inteiros no intervalo de -32.768 a 32.767 (ou 0 a 65.535 se for sem sinal).

  • INT = Geralmente armazena 4 bytes (32 bits), permitindo representar números inteiros no intervalo de -2.147.483.648 a 2.147.483.647 (ou 0 a 4.294.967.295 se for sem sinal).

  • BIGINT = Geralmente armazena 8 bytes (64 bits), permitindo representar números inteiros no intervalo de -9.223.372.036.854.775.808 a 9.223.372.036.854.775.807 (ou 0 a 18.446.744.073.709.551.615 se for sem sinal).

  • NUMERIC/DECIMAL = A quantidade de bytes que um tipo decimal ocupa depende do número total de dígitos e do número de dígitos à direita do ponto decimal que você escolher. Em sistemas de banco de dados, a precisão e a escala são usadas para especificar esses detalhes:

CARACTERES

  • CHAR = Armazena dados adicionados por String de forma que, independente do valor real da String, ele armazenará o valor alocado a ele.

  • VARCHAR = Diferente do CHAR, ao adicionar um valor real menor do que o definido para ele, ele reduz, armazenando apenas o que entregue pela String.

RESTRIÇÕES

São códigos SQL que você irá restringir dados que forem adicionados dentro da coluna.

  • PRIMARY KEY = Uma restrição que garante que os valores em uma ou mais colunas de uma tabela sejam únicos e não nulos. A chave primária identifica exclusivamente cada registro em uma tabela.

  • NOT NULL = Faz com que não seja possível deixar a coluna em branco, exijindo sempre um dado com base na especificação dela.

  • AUTO INCREMENT = Faz com que a coluna seja preenchina automaticamente com base na espeficação dela.

  • FOREIGN KEY = Uma restrição que define um relacionamento entre duas tabelas. Ela garante que os valores em uma coluna (a chave estrangeira) correspondam aos valores em outra tabela (a chave primária ou outra chave única).

  • UNIQUE = Uma restrição que garante que todos os valores em uma coluna (ou um conjunto de colunas) sejam únicos. Essa restrição permite valores nulos, mas apenas um valor nulo é permitido.

  • CHECK = Uma restrição que verifica se os valores em uma coluna satisfazem uma condição específica. Por exemplo, você pode definir uma restrição de verificação para garantir que os valores em uma coluna numérica sejam maiores que zero.

OUTRAS OPÇÕES

  • DEFAULT = Uma restrição que define um valor padrão para uma coluna, que será usado quando nenhum valor for especificado durante a inserção de um novo registro.

  • INDEX = Uma restrição que cria um índice para uma ou mais colunas de uma tabela, permitindo consultas mais eficientes ao acessar os dados dessa coluna.

  • CASCADE = Uma opção que pode ser usada em conjunto com restrições como DELETE ou UPDATE, especificando que as ações de exclusão ou atualização em uma tabela devem ser propagadas para tabelas relacionadas.

  • DISABLE/ENABLE = Restrições que podem ser desabilitadas temporariamente e depois reabilitadas. Isso pode ser útil durante operações de manutenção ou migração de dados.

  • DEFERRABLE = Uma restrição que pode ser adiada até o final de uma transação, permitindo que as restrições sejam verificadas apenas no final da transação.


CREATE DATABASE

O comando CREATE DATABASE é usado para criar um novo banco de dados.

        CREATE DATABASE nome_do_banco_de_dados;

CREATE TABLE

O comando CREATE TABLE é usado para criar uma nova tabela em um banco de dados existente.

        CREATE TABLE nome_da_tabela (
            id INT PRIMARY KEY,
            nome VARCHAR(50),
            idade INT
        );

INSERT INTO

Comando utilizado para inserir algum dado nas colunas da tabela.

    
        INSERT INTO nome_da_tabela (usuario, senha)
        VALUES ("usuario", "senha");

Nesse exemplo o administrador adicionou um novo usuário e senha apenas na tabela.

UPDATE

O comando UPDATE é usado para atualizar um dado já cadastrado na coluna da tabela.

        UPDATE nome_da_tabela
                SET nome_da_coluna = NovoDado

Se na tabela tiver mais de 1 dado, você pode selecionar o dado que quer alterar utilizando como base, por exemplo, o id.

        UPDATE nome_da_tabela
                SET nome_da_coluna = NovoDado
                WHERE id = 1;

Nesse exemplo ele irá trocar apenas os dados da coluna que estiver com o id = 1.
Caso você queira altera mais de um dado em apenas uma linha de código, basta separar por vírgulas.

        UPDATE nome_da_tabela
                SET usuario = "novoNomeUsuario", senha = "novaSenha"
                WHERE id = 1;

Nesse caso ele alterou o nome de usuário e a senha da pessoa com ID 1 dentro do banco de dados.
